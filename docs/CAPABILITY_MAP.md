## 实现边界

- 本文档只记录功能边界、行为类别、验收标准和 FetchDock 自有实现方案。
- 不把第三方项目的源码、图片、图标、商标、角色形象、截图、README 文案或独有视觉设计纳入 FetchDock。
- 若未来确需引入第三方代码，必须在对应模块标注来源与许可证，并让项目遵守对应许可证；默认路线为不复制源码的自写实现。
- 不新增已列入能力之外的产品能力；AI 相关能力只在当前规格明确存在时列入。
- 所有状态从 `todo` 开始；只有完成实现并通过验收后才能改为 `done`。无法实现或缺乏公开规格时标为 `blocked` 并写明原因。

## 规格输入

- 桌面应用范围：下载入口、本地队列、播放器/阅读器、课程、音乐、设置、插件、全局快捷键、便携模式、发布与法务。
- 浏览器扩展范围：页面交接、媒体候选捕获、本地 bridge、配对令牌、deep link fallback、权限边界、平台识别、打包与测试范围。
- 插件系统范围：动态库 manifest、ABI 检查、host API、插件命令、事件、设置、i18n、数据目录、registry 与 marketplace。
- 发布质量范围：插件兼容性保护、下载进度、路径捕获、HLS/referer、portable mode、CLI、扩展配对和平台修复记录。
- 业务插件范围：课程、Telegram、转换工具和后续本地扩展能力。
